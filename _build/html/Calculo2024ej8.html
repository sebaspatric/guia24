Pregunta8#

Problema 8: Aproximaci√≥n de una integral con la regla del trapecio#

Utilice la regla del trapecio con \(n = 5\) para aproximar la siguiente integral:

\[ I = \int_{-1/2}^{1/2} e^{-x^2} \, dx \]

Adem√°s, determine si dicha aproximaci√≥n es por exceso o por defecto.

Desarrollo#

Aproximar la integral

\[ I = \int_{-1/2}^{1/2} e^{-x^2} \, dx \]

usando la regla del trapecio con \(n=5\) subintervalos.

Paso 1: Regla del trapecio#

La regla del trapecio para \(n\) subintervalos es:

\[ \int_a^b f(x) dx \approx \frac{h}{2} \left[ f(x_0) + 2 \sum_{i=1}^{n-1} f(x_i) + f(x_n) \right] \]

donde

  • \(h = \frac{b-a}{n}\) es el ancho de cada subintervalo,

  • \(x_i = a + i h\), \(i=0,\dots,n\).

Paso 2: Par√°metros y puntos#

Dado:

  • \(a = -\frac{1}{2} = -0.5\),

  • \(b = \frac{1}{2} = 0.5\),

  • \(n = 5\).

Calculamos el ancho:

\[ h = \frac{0.5 - (-0.5)}{5} = \frac{1}{5} = 0.2 \]

Los nodos son:

\[\begin{split} \begin{aligned} x_0 &= -0.5 \\ x_1 &= -0.3 \\ x_2 &= -0.1 \\ x_3 &= 0.1 \\ x_4 &= 0.3 \\ x_5 &= 0.5 \end{aligned} \end{split}\]

Paso 3: Evaluaci√≥n de la funci√≥n \(f(x) = e^{-x^2}\)#

Calculamos:

\[\begin{split} \begin{aligned} f(x_0) &= e^{-(-0.5)^2} = e^{-0.25} \approx 0.7788 \\ f(x_1) &= e^{-(-0.3)^2} = e^{-0.09} \approx 0.9139 \\ f(x_2) &= e^{-(-0.1)^2} = e^{-0.01} \approx 0.9900 \\ f(x_3) &= e^{-(0.1)^2} = e^{-0.01} \approx 0.9900 \\ f(x_4) &= e^{-(0.3)^2} = e^{-0.09} \approx 0.9139 \\ f(x_5) &= e^{-(0.5)^2} = e^{-0.25} \approx 0.7788 \end{aligned} \end{split}\]

Paso 4: Aplicar la regla del trapecio#

\[ I \approx \frac{h}{2} \left[ f(x_0) + 2(f(x_1) + f(x_2) + f(x_3) + f(x_4)) + f(x_5) \right] \]

Sustituimos:

\[ I \approx \frac{0.2}{2} \left[ 0.7788 + 2(0.9139 + 0.9900 + 0.9900 + 0.9139) + 0.7788 \right] \]

Calculamos la suma interna:

\[ 0.9139 + 0.9900 + 0.9900 + 0.9139 = 3.8078 \]

Multiplicamos por 2:

\[ 2 \times 3.8078 = 7.6156 \]

Sumamos extremos:

\[ 0.7788 + 7.6156 + 0.7788 = 9.1732 \]

Multiplicamos por \(\frac{h}{2} = 0.1\):

\[ I \approx 0.1 \times 9.1732 = 0.9173 \]

Paso 5: Valor exacto usando la funci√≥n error#

La funci√≥n \(e^{-x^2}\) no tiene antiderivada elemental, pero se relaciona con la funci√≥n error:

\[ \operatorname{erf}(x) = \frac{2}{\sqrt{\pi}} \int_0^x e^{-t^2} dt \]

De aqu√≠,

\[ \int_0^a e^{-t^2} dt = \frac{\sqrt{\pi}}{2} \operatorname{erf}(a) \]

Por ser \(e^{-x^2}\) una funci√≥n par,

\[ \int_{-a}^a e^{-x^2} dx = 2 \int_0^a e^{-x^2} dx = \sqrt{\pi} \operatorname{erf}(a) \]

Con \(a = 0.5\):

\[ I = \sqrt{\pi} \operatorname{erf}(0.5) \]

Paso 6: C√°lculo num√©rico del valor exacto#

Sabemos:

  • \(\sqrt{\pi} \approx 1.77245385\),

  • \(\operatorname{erf}(0.5) \approx 0.52049988\).

Entonces:

\[ I \approx 1.77245385 \times 0.52049988 = 0.922137 \]

Paso 7: Comparaci√≥n y an√°lisis de error#

Valor aproximado (trapecio)

Valor exacto (funci√≥n error)

Diferencia

0.9173

La funci√≥n \(f(x)\) es c√≥ncava hacia abajo en \([-0.5,0.5]\), por lo que la regla del trapecio subestima la integral. Esto coincide con que el valor aproximado es menor que el valor exacto.

Conclusi√≥n final#

\[\begin{split} \boxed{ \begin{cases} \text{Aproximaci√≥n por regla del trapecio} = 0.9173 \\ \text{Valor exacto (num√©rico)} = 0.9221 \\ \text{Error: por defecto (subestimaci√≥n)} \end{cases} } \end{split}\]

¬øPor qu√© se usa?#

  1. No se puede encontrar una primitiva (antiderivada) exacta:

    • Algunas funciones, como \(e^{-x^2}\), no tienen una antiderivada elemental, por lo que no se puede integrar exactamente usando m√©todos anal√≠ticos.

    • En esos casos, recurrimos a m√©todos num√©ricos como el del trapecio, Simpson, o cuadratura gaussiana.

  2. Se necesita una estimaci√≥n r√°pida o computacional:

    • Si est√°s programando o modelando y necesitas obtener r√°pidamente un √°rea bajo la curva.

    • El m√©todo del trapecio es simple de implementar, incluso sin calculadora simb√≥lica.

  3. Cuando se tiene una tabla de datos en lugar de una f√≥rmula:

    • Si tienes solo ciertos puntos \((x_i, f(x_i))\), y no una funci√≥n expl√≠cita.

    • Por ejemplo, datos experimentales o mediciones f√≠sicas.

¬øCu√°ndo se aplica?#

Aplica mejor cuando:

  • La funci√≥n es suave y continua en el intervalo.

  • La funci√≥n no oscila mucho (no tiene muchas curvaturas).

  • El intervalo se puede dividir en partes iguales f√°cilmente.

Se aplica a integrales definidas del tipo:

\[ \int_a^b f(x) \, dx \]

donde:

  • \(f(x)\) est√° definida y es continua en \([a, b]\)

  • Se divide el intervalo \([a, b]\) en \(n\) subintervalos.

  • Se construyen trapecios entre los puntos para aproximar el √°rea bajo la curva.

Ventajas y desventajas#

Ventajas

Desventajas

F√°cil de entender y programar

Menor precisi√≥n que otros m√©todos m√°s avanzados (como Simpson)

Funciona con datos discretos

Requiere muchos intervalos para funciones muy curvas

√ötil cuando no hay antiderivada anal√≠tica

Puede tener error significativo si la funci√≥n es muy c√≥ncava o convexa

En resumen#

El m√©todo del trapecio se aplica cuando:

  • No se puede integrar simb√≥licamente una funci√≥n.

  • Se tienen datos discretos (no una f√≥rmula expl√≠cita).

  • Se busca una estimaci√≥n r√°pida y razonablemente precisa del valor de una integral definida.

import numpy as np
import plotly.graph_objects as go

# Funci√≥n a integrar
def f(x):
    return np.exp(-x**2)

a, b = -0.5, 0.5
x_fine = np.linspace(a, b, 1000)
y_fine = f(x_fine)
area_real = np.trapz(y_fine, x_fine)
N_max = 10

fig = go.Figure()

# √Årea real
fig.add_trace(go.Scatter(
    x=x_fine, y=y_fine, fill='tozeroy',
    line=dict(color='blue'), opacity=0.2, name='√Årea real'
))

# Nodos conectados
fig.add_trace(go.Scatter(
    x=[], y=[], mode='lines+markers',
    line=dict(color='red', width=2),
    marker=dict(color='red', size=8),
    name='Nodos'
))

# Textos de trapecios
fig.add_trace(go.Scatter(
    x=[], y=[], mode='text', text=[], textposition='top center',
    showlegend=False
))

# √Årea total
fig.add_trace(go.Scatter(
    x=[(a+b)/2], y=[max(y_fine)*1.2], mode='text', text=[""], 
    showlegend=False, textfont=dict(size=14, color='black')
))

# Precrear N_max traces para trapecios
for _ in range(N_max):
    fig.add_trace(go.Scatter(
        x=[], y=[], fill='toself', fillcolor='orange', opacity=0.3,
        line=dict(color='orange'), showlegend=False, name='Trapecio'
    ))

# Crear steps del slider
steps = []
for n in range(1, N_max+1):
    x_nodes = np.linspace(a, b, n+1)
    y_nodes = f(x_nodes)
    
    # Trapecios
    trap_texts = []
    text_x = []
    text_y = []
    area_total = 0
    trap_x = []
    trap_y = []

    for i in range(n):
        x0, x1 = x_nodes[i], x_nodes[i+1]
        y0, y1 = y_nodes[i], y_nodes[i+1]
        area_trap = 0.5*(y0 + y1)*(x1 - x0)
        area_total += area_trap
        xs = [x0, x0, x1, x1, x0]
        ys = [0, y0, y1, 0, 0]
        trap_x.append(xs)
        trap_y.append(ys)
        text_x.append((x0+x1)/2)
        text_y.append((y0+y1)/2)
        trap_texts.append(f"{area_trap:.4f}")

    # L√≠neas divisorias
    shapes = [dict(
        type="line",
        x0=xi, x1=xi,
        y0=0, y1=max(y_fine)*1.35,
        line=dict(color="black", width=2, dash="dot"),
        layer="above"
    ) for xi in x_nodes]

    # Step del slider
    step = dict(
        method="update",
        args=[{
            "x": [x_fine, x_nodes, text_x, [(a+b)/2]] + trap_x + [[]]*(N_max-n),
            "y": [y_fine, y_nodes, text_y, [max(y_fine)*1.2]] + trap_y + [[]]*(N_max-n),
            "text": [
                [f"√Årea real ‚âà {area_real:.4f}"]*len(x_fine),
                None,
                trap_texts,
                [f"√Årea total trapecios ‚âà {area_total:.4f}"]
            ] + [None]*N_max
        },
        {"shapes": shapes}],
        label=str(n)
    )
    steps.append(step)

# Slider
sliders = [dict(
    active=4,
    currentvalue={"prefix": "N√∫mero de trapecios: "},
    pad={"t":50},
    steps=steps
)]

# Layout final
fig.update_layout(
    autosize=True,
    width=None,
    sliders=sliders,
    title="√Årea bajo f(x) ‚âà exp(-x¬≤) usando regla del trapecio",
    xaxis_title="x",
    yaxis_title="f(x)",
    yaxis=dict(range=[0, max(y_fine)*1.4]),
    height=None,
    template="plotly_white",
    annotations=[dict(
        x=0.05, y=1.08, xref="paper", yref="paper",
        text=f"√Årea real ‚âà {area_real:.4f}", showarrow=False,
        font=dict(size=14, color="blue")
    )]
)

# Guardar HTML
fig.write_html(
    "_build/html/trapecio.html",
    include_plotlyjs="cdn",
    full_html=False,
    auto_open=False
)

fig.show()
